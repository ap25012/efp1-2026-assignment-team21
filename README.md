Τίτλος Συστήματος:
Σύστημα Προσφοράς Προϊόντων & Υπηρεσιών

1. Περιγραφή
Σύντομη περιγραφή των σεναρίων που υλοποιεί η ομάδα:

Ποιος είναι ο βασικός σκοπός του συστήματος:
Σκοπός του συστήματος είναι η διάθεση μεταχειρισμένων προϊόντων και η παροχή υπηρεσιών. Η πλατφόρμα συνδέει τους χρήστες-δωρητές και τους παρόχους με χρήστες που έχουν ανάγκη,
ενώ παράλληλα επιβραβεύει την προσφορά μέσω ενός συστήματος πόντων, δίνοντας έτσι κίνητρο στους δωρητές.

Ποιοι είναι οι βασικοί χρήστες (ρόλοι):
Οι βασικοί χρήστες(ρόλοι) του συστήματος είναι: 
Ο Χρήστης που Λειτουργεί ως Δωρητής (καταχωρεί προϊόντα, κερδίζει πόντους). 
Ο Πάροχος Υπηρεσίας που καταχωρεί διαθέσιμες υπηρεσίες με μια σύντομη περιγραφή και τον χρόνο.


Τα use cases υλοποιούμε από άκρη σε άκρη είναι:
Καταχώρηση Προϊόντος(Δωρεάς): Ο χρήστης κάνει login/sign in και προσφέρει κάποιο/α μεταχειρισμένο/α προϊόν/ ντα. Για κάθε
προϊόν επιλέγει κατηγορία (πχ. οικιακή συσκευή, ρούχο, έπιπλο) και εισάγει έτος_αγοράς, έτη
χρήσης, ονομασία, μάρκα (αν υπάρχει) και φωτογραφία. Για κάθε δωρεά του, ο χρήστης παίρνει
10 πόντους. 
Καταχώρηση Υπηρεσίας: Ο πάροχος υπηρεσίας κάνει login/sign in και επιλέγει κατηγορία υπηρεσίας (πχ.
τεχνική εργασία) και εισάγει ονομασία και σύντομη περιγραφή υπηρεσίας και διαθέσιμο χρόνο /
υπηρεσία (πχ. 1 ώρα τη φορά).

5. Δομή του κώδικα
Σύντομη περιγραφή των αρχείων:

efp1-2026-assignment-team21/    # Route του project
.
├── main.py         # Είσοδος του προγράμματος / μενού / διάλογος με χρήστη
├── models.py       # Κλάσεις τομέα (π.χ. User, Request, Document, κ.λπ.)
├── services.py     # Λογική συστήματος / συναρτήσεις χειρισμού σεναρίων
├── login.py        # Είναι υπεύθυνο για την σύνδεση-ταυτοποίηση του παρόχου ή του χρήστη. 
└── README.md       # Οδηγίες
└── Happy_Path.txt  # Σενάριο εκτέλεσης και επεξήγηση διαγραμμάτων
└── diagrams        # Διαγράμματα
    ├── Class Diagram_Μπαρδάκη.jpg
    ├── Class Diagram_Φραγκιαδάκης.jpg
    ├── Use Case Diagram_Μπαρδάκη.jpg
    └── Use Case Diagram_Φραγκιαδάκης.jpg
    └── ER_Diagram_Φραγκιαδάκης.jpg
    └── Καταχώρηση μεταχειρισμένου προϊόντος Sequence Diagram.jpg
    └── Καταχώρηση υπηρεσίας Sequence Diagram.jpg
    
 
Περιγραφή λειτουργίας του κάθε αρχείου:
main.py: περιέχει το μενού,διαβάζει input από τον χρήστη,καλεί τις συναρτήσεις από services.py.
models.py: περιέχει τις βασικές κλάσεις του συστήματος (οντότητες).
services.py: περιέχει συναρτήσεις που υλοποιούν τα βήματα των use cases (δημιουργία/αναζήτηση/ενημέρωση αντικειμένων).
login.py: αναλαμβάνει αποκλειστικά την ταυτοποίηση και σύνδεση των χρηστών ή παρόχων.
README.md: το αρχείο με τις βασικές οδηγίες εγκατάστασης και εκτέλεσης του προγράμματος.
Happy_Path.txt: η περιγραφή της ιδανικής εκτέλεσης του προγράμματος (χωρίς σφάλματα) και η επεξήγηση των διαγραμμάτων.
diagrams: ο φάκελος με τα διαγράμματα, class_diagrams για τη δομή των κλάσεων, sequence_diagrams για τη σειρά των ενεργειών, ER_diagram για τις συσχετίσεις των οντοτήτων και use_case diagrams για τις βασικές ενέργειες.
